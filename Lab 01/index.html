Definiciones

URL:
Sus siglas significan "Uniform Resource Locator". Es la dirección de una página web específica o archivo en internet. [1]
MIME:
Significa Multi-Purpose Internet Mail Extensions. Como su nombre lo indica, es una extension al protocolo de correo electrónico, permite al usuario intercambiar archivos a través del internet, como imágenes, audio y video. [2]
Métodos y códigos de error HTTP:
HTTP define un conjunto de métodos de solicitud para indicar la acción deseada para un recurso dado. A éstos métodos se les llama vebos HTTP. [3]
HTML5:
Es el estándar más reciente de HTML (Hypertext Markup Language). [4]
JS:
JavaScript es un lenguaje de programación que permite implementar características complejas en páginas web. Es decir, maneja el comportamiento de una página web. [5]
PHP:
PHP: Hypertext Processor es un lenguaje de programación adecuado para desarrollo web y que puede ser embebido en HTML. [6]
JSP:
JavaServer Pages es una tecnología estándar de Java que permite escibir páginas dinámicas para aplicaciones web de Java. [7]
AJAX:
Asynchronous JavaScript And XML es un conjunto de técnicas para construir sitios y aplicaciones web. La función principal de AJAX es actualizar contenido web sin necesidad de refrescar el navegador. [8]

Preguntas a Responder

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es una red de redes global, una conección entre millones de computadoras, servidores, y dispositivos. Mientras que la World Wide Web es un sistema estandarizado para accesar y navegar en internet.
Visto de otra forma, el internet es la infraestructura, y la World Wide Web es un servicio que utiliza dicha infraestructura. [9]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET:
El método GET se usa para "leer" o recuperar la representación de un recurso. Retorna un archivo XML o JSON, además de un código de respuesta HTTP de 200 (OK). EN caso de un error, regresa un código 404 (NOT FOUND) o 400 (BAD REQUEST). [10]
Ejemplos:
  • GET http://www.ejemplo.com/123
  • GET http://www.ejemplo/seccion/123
  • GET http://www.ejemplo/seccion/articulo
HEAD:
Es idéntico al método GET, con la excepción de que debe retornar únicamente el encabezado en la respuesta. [10]
POST:
Se utiliza por lo general para crear nuevos recursos. En particular, para crear recursos subordinados a un recurso "padre". Cuando se crea este nuevo recurso, retorna un código de 201 (Created), junto con un header de ubicación con un link al nuevo recurso. [10]
Ejemplos:
  • POST http://www.ejemplo.com/seccion
  • POST http://www.ejemplo.com/seccion/articulo
PUT:
Se utiliza para actualizar recursos. Es decir, modificar un recurso creado previamente con los nuevos datos. Cuando se actualiza exitosamente un recurso, retorna un código de 200 (OK). [10]
Ejemplos:
  • PUT www.ejemplo.com/nueva_seccion
  • PUT www.ejemplo.com/seccion/nuevo_articulo
PATCH:
Este método se usa para capacidades de modificación. Sólo se necesita contener los cambios al recurso, no todo el recurso. Sin embargo, el cuerpo no sólo debe ser la parte modificada del recurso, tiene que estar en algún lenguaje de parcheo, como JSON Patch o XML Patch. [11]
DELETE:
Como su nombre lo indica, se usa para eliminar un recurso.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

Si tomamos como referencia el "Modelo en cascada":

  1. Planificación
  2. Análisis
  3. Diseño
  4. Implementación
  5. Pruebas
  6. Mantenimiento [12]

¿Cuál es el ciclo de desarrollo de sistemas de información?

  1. Identificación de problemas, oportunidades y objetivos.
  2. Determinación de requerimientos.
  3. Análisis de necesidades del sistema.
  4. Diseño del sistema.
  5. Desarrollo y documentación.
  6. Pruebas y mantenimiento.
  7. Implantación y evaluación del sistema. [12]
